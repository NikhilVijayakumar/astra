# 📐 Astra: Atomic Rules & Structural Logic

> **Version:** 1.0.0
> **Status:** Mandatory Enforcement for Antigravity Agent Synthesis

## 1. The Atomic Hierarchy

All UI elements must belong to their appropriate layer. Each component file should operate at one layer — a file should not mix the concerns of an Atom with the layout responsibilities of an Organism. A page template will naturally contain Organisms, Molecules, and Atoms, but each of those elements must be authored and maintained at its own layer boundary.

| Layer | Definition | MUI Mapping |
| --- | --- | --- |
| **Atoms** | Smallest functional units (Icons, Typography, Buttons). | `MuiButton`, `MuiTypography`, `MuiIcon` |
| **Molecules** | Groups of Atoms functioning together (Search Bar, Agent Status Chip). | `MuiBox` (with children), `MuiCardHeader` |
| **Organisms** | Complex UI sections (Navigation Sidebar, Table Header with Filters). | `MuiAppBar`, `MuiDrawer`, `MuiDataGrid` |
| **Templates** | Full page structures that assemble Organisms into layout shells. | `Container`, `PageHeader`, `SummaryPanel`, `HeroSection` |

---

## 2. Structural Constraints (The 8px Rule)

To ensure the separate **Verification Project** passes the layout audit, the following spacing rules are absolute:

* **The Grid:** All `padding`, `margin`, and `gap` values must be multiples of **8px**.
* **CSS Variables:** Never use hardcoded pixel values (e.g., `15px`). Always use foundations: `var(--mui-spacing-1)` (8px), `var(--mui-spacing-2)` (16px), etc.
* **Flexbox Geometry:**
* Molecules must use `display: flex` or `display: grid`.
* Standardized `gap` is `var(--mui-spacing-2)` unless otherwise specified in feature docs.



---

## 3. Typography Enforcement

The Agent must map all text to the predefined typography variants in `foundations/typography.css`.

* **Headings:** Use `h1` through `h6` for structural hierarchy.
* **Body:** Use `body1` for standard text and `body2` for secondary/dense information.
* **Hero:** Reserved for high-impact dashboard values only.
* **Fluidity:** Do not override `font-size`. Rely on the `clamp()` logic defined in the foundations to handle HD vs 4K viewports.

---

## 4. Theme-Aware Rendering (No Color Hardcoding)

The Agent is forbidden from using hex codes (`#FFFFFF`) or literal colors (`red`).

* **Logic:** Every color must be a theme variable.
* **Primary Action:** `var(--mui-primary-main)`
* **Surface:** `var(--mui-bg-paper)`
* **Text:** `var(--mui-text-primary)` or `var(--mui-text-secondary)`
* **Status Colors:** Use semantic variables: `--mui-success-main`, `--mui-error-main`, `--mui-warning-main`.

---

## 5. Navigation & Logic (The "Intent" Rule)

If a feature document mentions a click or interaction, the Agent must:

1. Add a `data-navigation-intent` attribute to the HTML element.
2. Populate the `nav_logic` key in the `metadata.json`.
3. **Strict Rule:** No `<a>` tags with hardcoded URLs. Use interaction intents (e.g., `intent: "route_to_agent_detail"`).

---

## 6. Statelessness (Mockup Purity)

Mockups generated by Astra are **Stateless Blueprints**.

* **No `<script>` tags** inside `markup.html` for business logic.
* **Interactivity:** Use CSS `:hover`, `:active`, and `:focus` states only.
* **Dynamic Data:** Use `{{handlebars}}` style placeholders that map to the `locals/*.json` files.

---

## 7. Viewport Specifics

* **Desktop (1080p - 4K):** Prioritize information density and horizontal layouts.
* **Mobile/Tablet:** Prioritize vertical stacking and touch-friendly targets (minimum 44px height).
