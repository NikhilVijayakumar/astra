# Astra — Premium UI Patterns

Premium interfaces are defined by **clarity, hierarchy, and restraint**.
Astra UI patterns ensure that every screen communicates information **efficiently and elegantly**.

These patterns should guide the design of dashboards, tools, and product interfaces.

---

# 1. Card Surfaces

![Image](https://www.denovers.com/_next/image?q=75\&url=https%3A%2F%2Fcdn.sanity.io%2Fimages%2Fw43feo7g%2Fproduction%2Ffe5c6fc45f383ac257e9d80ae97fefc6c834f9b2-1600x800.webp%3Ffit%3Dmax%26auto%3Dformat\&w=1600)

![Image](https://miro.medium.com/v2/resize%3Afit%3A1400/1%2AFAmhxwwj3kL5X-TjMHbKTA.jpeg)

![Image](https://cdn.sanity.io/images/hkc8ojqt/production/9acbde48b49cdec2c85ad41567a673ee70f25aa6-1920x1008.jpg)

![Image](https://tailadmin.com/_next/image?q=75\&url=https%3A%2F%2Fcdn.sanity.io%2Fimages%2Fhkc8ojqt%2Fproduction%2Fe8c7251a9eed63a9bb80403d7becaca5b694c5d0-1800x900.png%3Ffit%3Dmax%26auto%3Dformat\&w=3840)

Cards organize information into **clear, digestible units**.

### Characteristics

• soft surface contrast
• subtle elevation
• comfortable internal spacing
• clear content hierarchy

### Card Content Structure

A card should typically contain:

1. Title
2. Supporting information
3. Primary action or data

### Guidelines

Cards should never feel:

* cramped
* overly decorated
* visually heavy

**Principle:**
Cards are containers for clarity, not decoration.

![Image](https://mir-s3-cdn-cf.behance.net/project_modules/fs/2a3bf0105080223.5f7eb903dd0c3.jpg)

Hero sections introduce a page with **confidence and clarity**.

### Characteristics

• large confident typography
• generous whitespace
• minimal supporting elements

### Typical Structure

1. Headline
2. Short supporting description

Avoid:

* cluttered hero sections
* multiple competing messages
* heavy graphics

**Principle:**
One clear message is more powerful than many.

# 3. Structured Data Tables

![Image](https://cdn.dribbble.com/userupload/46911894/file/4593251ca7a9d81674219e524ddba304.png?format=webp\&resize=400x300\&vertical=center)

![Image](https://cdn.dribbble.com/userupload/15533941/file/original-228150e24d717be429aec34691fc9d9e.png?format=webp\&resize=400x300\&vertical=center)

Data tables should prioritize **legibility and scanning speed**.

### Characteristics

• clear column alignment
• subtle row separators
• readable spacing
• minimal borders

### Interaction Guidelines

Tables should support:

* sorting
* filtering
* quick scanning

Avoid:

* dense rows
* visual clutter
* excessive grid lines

**Principle:**
Data should feel structured and easy to read.

# 4. Action-Focused Toolbars

![Image](https://cdn.dribbble.com/userupload/14108850/file/original-cbfb5c51eccf06c66234dc5ddfc0aeba.png?format=webp\&resize=400x300\&vertical=center)

![Image](https://cdn.dribbble.com/userupload/14640639/file/original-9c2b8d3cd71840bf14e3ca8a883b672f.png?format=webp\&resize=400x300\&vertical=center)

![Image](https://cdn.dribbble.com/userupload/44376881/file/59e8ce110f5d57e039be3d08adf5bd71.png?resize=752x\&vertical=center)

![Image](https://s3-alpha.figma.com/hub/file/2300310130347608777/7c660eac-fb7a-4074-a7ec-de135aa6d90f-cover.png)

Toolbars provide **quick access to key actions**.

### Characteristics

• clear grouping of actions
• minimal icon usage
• balanced spacing

Primary actions should be clearly visible without overwhelming the interface.

Avoid:

* too many icons
* ambiguous actions
* inconsistent grouping

**Principle:**
Tools should support flow, not interrupt it.

• logical grouping

# 6. Contextual Side Panels

![Image](https://cdn.dribbble.com/userupload/20584412/file/original-a3c35ecbe32a5a24eaefb769e27a21f7.png?format=webp\&resize=400x300\&vertical=center)

![Image](https://cdn.dribbble.com/userupload/11532132/file/original-39bc04af27be78ab27693e9a65707f6c.jpg?resize=400x0)

![Image](https://cdn.trendhunterstatic.com/thumbs/530/minimalist-productivity-app.jpeg)

![Image](https://s3-alpha.figma.com/hub/file/2300310130347608777/7c660eac-fb7a-4074-a7ec-de135aa6d90f-cover.png)

Side panels allow users to **view or edit details without leaving context**.

### Characteristics

• smooth sliding transitions
• clear section hierarchy
• focused content

Typical uses:

* item details
* settings
* quick editing

Avoid:

* overly wide panels
* unrelated content
* cluttered layouts

**Principle:**
Panels preserve context and reduce navigation friction.

![Image](https://cdn.dribbble.com/userupload/17374188/file/original-69b8db1921d53118b23dd7bc7332a096.png?format=webp\&resize=400x300\&vertical=center)

Notifications provide feedback without disrupting the user.

### Characteristics

• short and clear messages
• minimal styling
• automatic dismissal

Avoid:

* large intrusive alerts
* unnecessary animations
* overly bright colors

**Principle:**
Feedback should be helpful, not distracting.

# Premium UI Pattern Philosophy

Astra interface patterns are defined by:

**Clarity**
Information should be immediately understandable.

**Structure**
Layouts should feel engineered and balanced.

**Restraint**
Visual design should remain minimal and intentional.
