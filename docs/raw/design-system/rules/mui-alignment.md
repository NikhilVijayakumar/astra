# React & MUI Alignment Guidelines

## Overview

The HTML mockups generated by the Astra pipeline serve as the foundation for the final React application. To ensure the final codebase scales effectively and leverages Material UI (MUI) v7 and the Astra component library properly, developers and AI agents must follow this strict alignment guide when translating static HTML mockups into React.

---

## 1. Core Composition Patterns

Borrowing from advanced React architecture (e.g., Vercel Composition Patterns), we fundamentally reject "boolean prop proliferation."

### 1.1 Avoid Boolean Props for Variations
**INCORRECT (The Boolean Trap):**
```tsx
// ❌ Don't do this. It leads to bloated components.
<PremiumCard
  isDark={true}
  hasGlassmorphism={true}
  withStripeBorder={false}
>
  Content
</PremiumCard>
```

**CORRECT (Composition & Variants):**
```tsx
// ✅ Do this. Use explicit variants or compose behaviors.
<Surface variant="glass-dark">
  <CardContent>Content</CardContent>
</Surface>
```

### 1.2 Compound Components
When translating complex atomic HTML elements (like a `[data-section="global-header"]`), break them down into compound components that share context, rather than passing 15 props down a single monolith.

```tsx
<GlobalHeader>
  <GlobalHeader.Logo href="/home" />
  <GlobalHeader.Navigation>
    <NavItem intent="dashboard" />
  </GlobalHeader.Navigation>
  <GlobalHeader.Actions>
    <ThemeToggle />
  </GlobalHeader.Actions>
</GlobalHeader>
```

## 2. Default Target Architecture: MUI v7 & Astra Library

The static HTML maps predictably to idiomatic React components utilizing MUI v7 and the Astra component library.

| Astra HTML / CSS Concept | Idiomatic React Component | Notes |
| :--- | :--- | :--- |
| `display: flex; gap: 24px;` | `<Box sx={{ display: 'flex', gap: ... }}>` | Avoid nested flex boxes when a simple `<Box>` or `<Stack>` suffices. |
| `display: grid; ...` | `<Box sx={{ display: 'grid' }}>` | Align strictly with MUI v7 conventions for layouts. |
| `<div class="comp-surface">`| `<Paper elevation={X}>` | Map the shadow intensity to `elevation`. |
| `<h1 class="typography-h1">`| `<Typography variant="h1">` | Ensure the `brand-identity.yaml` theme parameters are passed via the `<ThemeProvider>`. |
| `Theme Toggling Logic`| `useTheme() from 'astra'` | The `astra` library handles global state like theme context. |
| `API Calling / Fetching`| `ApiService from 'astra'` | Map data-fetching intents to `astra` `ApiService` and `ServerResponse` structures. |

---

## 3. Theming & State Injection

### 3.1 Strict Theme Usage
Never hardcode hex colors or pixel values in `sx` props. All values must reference the MUI Theme object, which is generated directly from the portable `brand-identity.yaml`.

**INCORRECT:**
```tsx
<Box sx={{ backgroundColor: '#141415', borderRadius: '16px' }}>
```

**CORRECT:**
```tsx
<Box sx={{ bgcolor: 'background.paper', borderRadius: 4 }}> // Assuming radius unit is 4px
```

### 3.2 Translating Intents to Actions
The mockups embed `data-navigation-intent="[Intent]"`. In React, these intents must be translated into mapped actions or Zustand dispatches.

```tsx
// HTML Mockup Element:
// <button data-navigation-intent="open-settings-modal">Settings</button>

// React Implementation:
const { openModal } = useUIStore();
<Button onClick={() => openModal('settings')}>Settings</Button>
```
