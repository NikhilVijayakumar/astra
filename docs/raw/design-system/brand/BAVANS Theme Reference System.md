All references are **publicly accessible** for verification.

## 2️⃣ Dark Theme — Primary Brand Expression

**Purpose:** Canonical experience, brand authority

### Color Direction

* Deep Navy / Soft Black backgrounds
* Warm off-white text
* Gold / muted neon accents (sparingly)

### Verified References (Dark Mode)

| Reference                                                                          | What to observe                   |
| ---------------------------------------------------------------------------------- | --------------------------------- |
| [https://activetheory.net](https://activetheory.net)                               | Tech-noir motion + typography     |
| [https://linear.app](https://linear.app)                                           | Dark UI clarity & contrast        |
| [https://www.apple.com/apple-vision-pro/](https://www.apple.com/apple-vision-pro/) | Premium dark storytelling         |
| [https://stripe.com/docs](https://stripe.com/docs)                                 | Readability in dark documentation |

### Non-Negotiables

* No pure black (`#000`)
* No pure white text
* Accents only for **meaning**, never decoration

---

## 3️⃣ Light Theme — Professional & Reading-First

**Purpose:** Accessibility, long-form consumption, institutional trust

### Color Direction

* Bone / parchment / warm stone backgrounds
* Charcoal / deep navy text
* Same accents as dark mode, desaturated

### Verified References (Light Mode)

| Reference                                            | What to observe                 |
| ---------------------------------------------------- | ------------------------------- |
| [https://studiorotate.com](https://studiorotate.com) | Editorial light design          |
| [https://twofold.studio](https://twofold.studio)     | Strategic whitespace            |
| [https://slack.com](https://slack.com)               | Accessible contrast & hierarchy |
| [https://notion.so](https://notion.so)               | Comfortable long-form reading   |

### Rules

* Never use pure white backgrounds
* Light mode ≠ washed-out mode
* Same structure as dark theme

## 4️⃣ Typography System (Most Important Layer)

Typography is the **voice of BAVANS**.

### Heading Typography

**Characteristics**

* Large scale
* High weight
* Tight tracking
* Editorial confidence

### Verified References

| Reference                                                      | What to study            |
| -------------------------------------------------------------- | ------------------------ |
| [https://www.apple.com/iphone/](https://www.apple.com/iphone/) | Large editorial headings |
| [https://stripe.com](https://stripe.com)                       | Typographic hierarchy    |
| [https://studiorotate.com](https://studiorotate.com)           | Bold minimal headlines   |

### Font Direction (Examples)

* Inter / Inter Display
* IBM Plex Sans
* Satoshi
* Neue Haas Grotesk (if licensed)

> Limit to **1 family**, max **2 weights sets**

---

### Body Typography

**Characteristics**

* Calm
* Neutral
* Easy to scan

### Verified References

| Reference                                                  | What to observe            |
| ---------------------------------------------------------- | -------------------------- |
| [https://stripe.com/docs](https://stripe.com/docs)         | Long-form clarity          |
| [https://notion.so](https://notion.so)                     | Comfortable line length    |
| [https://developer.apple.com](https://developer.apple.com) | Developer-friendly reading |

### Rules

* Line length: 60–75 characters
* Generous line height
* Never dense paragraphs

## 5️⃣ Spacing & Layout System

Whitespace = **trust + premium feel**

### Layout Philosophy

| Reference                                        | Why                  |
| ------------------------------------------------ | -------------------- |
| [https://www.apple.com](https://www.apple.com)   | Spatial storytelling |
| [https://linear.app](https://linear.app)         | Calm density         |
| [https://twofold.studio](https://twofold.studio) | Asymmetrical balance |

### Rules

* Prefer fewer sections, more space
* Asymmetry over rigid grids
* Avoid visual clutter at all costs

---

## 6️⃣ Motion & Micro-Interactions

Motion communicates **state**, not excitement.

### Motion References

| Reference                                            | What to study               |
| ---------------------------------------------------- | --------------------------- |
| [https://activetheory.net](https://activetheory.net) | Controlled cinematic motion |
| [https://www.apple.com](https://www.apple.com)       | System-level transitions    |
| [https://www.framer.com](https://www.framer.com)     | Micro-interaction patterns  |

### Rules

* Every animation must explain something
* No looping animations
* Ease > speed
* Motion must stop when user stops

---

## 8️⃣ Theme Parity Rules (Critical)

Dark & Light themes **must share**:

* Layout
* Typography scale
* Spacing
* Motion logic
* Interaction behavior

> Only **tokens change**, never structure.

# ✅ BAVANS Design QA Checklist (For Review & Sign-Off)

Use this checklist before **any release**.

---

## 🎨 Visual QA

* [ ] No pure black or pure white used
* [ ] Accent colors appear only for meaning
* [ ] Dark & Light themes look equally premium
* [ ] No visual clutter or filler graphics

---

## ✍️ Typography QA

* [ ] Headings feel editorial, not playful
* [ ] Body text readable for 10+ minutes
* [ ] Line lengths within range
* [ ] Consistent hierarchy across pages

---

## 📐 Spacing QA

* [ ] Sections breathe (not compressed)
* [ ] Whitespace feels intentional
* [ ] No crowded grids
* [ ] Visual rhythm consistent

---

## 🎞 Motion QA

* [ ] Every animation has purpose
* [ ] No unnecessary loops
* [ ] Motion stops when interaction stops
* [ ] Performance remains smooth on mid devices

---

## ♿ Accessibility QA

* [ ] WCAG 2.1 contrast compliance
* [ ] Keyboard navigation works
* [ ] Light theme readable in bright environments
* [ ] Dark theme comfortable for long sessions

---

## 🧠 Brand Integrity QA

* [ ] Feels like a **library**, not a toy
* [ ] Calm, confident, intelligent tone
* [ ] No exposure of internal logic or prompts
* [ ] Aligns with long-term BAVANS vision

---

## 🧭 Final Guidance for Antigravity

> If something looks impressive but distracting — remove it.
> If something feels empty but calm — keep it.
> If in doubt — default to **less**.
